## Application Under Test
UberEats Web Ordering Platform (Production Web Application)

## Project Type
Personal QA Portfolio Project  
Functional & Network Condition Testing on a live production system.
This project documents a personal quality assurance (QA) case study conducted on the UberEats web ordering platform.

## Testing Objective  
The objective was to evaluate the functional usability, reliability, and user experience of key customer-facing features, with focus on:

- Location selection and delivery address handling  
- Restaurant listing and selection
- Menu browsing  
- Add to Cart behavior  
- Cart management
- Pricing, totals, and fees calculation  
- Search functionality  
- Network throttling and offline handling  

## Scope of Testing
- Functional UI testing  
- Network condition simulation using Chrome DevTools  
- Basic resilience and error handling validation  

## Out of Scope
- Backend testing  
- Security testing  
- Payment processing  

## Tools Used
- Google Chrome DevTools (Network throttling, offline simulation)
- Microsoft Word (Test plan, Test scenerio)
- Microsoft Excel (Test case design and execution)  
- Jira (Bug reporting)  
- Loom (Test evidence recording)
